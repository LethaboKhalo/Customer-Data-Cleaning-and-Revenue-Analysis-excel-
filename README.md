# Customer Data Cleaning and Revenue Analysis (excel)
End-to-end customer data cleaning and analysis project in Excel. Standardized data, handled missing values, transformed dates, and generated business insights using Pivot Tables and statistical analysis.

## Overview
This project demonstrates an end-to-end data cleaning and analysis workflow using Microsoft Excel. The dataset contained missing values, inconsistent date formats, duplicate records, and non-standardized text entries that affected data quality and reporting accuracy.

The objective was to improve data quality, prepare the dataset for analysis, and generate business insights that could support data-driven decision-making.

## Business Problem
Organizations rely on accurate customer data to make informed decisions. Poor data quality can lead to incorrect reporting, unreliable insights, and inefficient business processes.

This project focuses on identifying and resolving common data quality issues before performing exploratory analysis

### Dataset
The dataset contains customer information including:

-Customer demographics
-Age
-Country
-Registration dates
-Spending information
-Customer notes and comments

## Data Cleaning Process

### Missing Values
-Identified missing values in the Age column
-Applied median imputation to replace missing ages
-Date Standardization
Converted multiple date formats into a consistent format, including:
2023-11-29T00:00:00Z
11/08/2023
April 19 2023
29-07-2023

### Text Cleaning
-Removed unwanted characters
-Standardized text formatting
-Corrected inconsistent values
### Duplicate Detection
-Checked for duplicate records
-Validated data integrity


## Analysis Performed

After cleaning the data, exploratory analysis was conducted to identify:
-Customer distribution by country
-Spending trends
-Average customer age
-Revenue patterns
-Customer demographics

#### Tools used:

Excel Functions
Pivot Tables
Conditional Formatting
Data Validation
Statistical Analysis

## Key Insights
-Identified spending patterns across countries
-Improved dataset completeness through imputation
-Standardized records for more reliable reporting
-Produced a clean dataset suitable for business analysis
